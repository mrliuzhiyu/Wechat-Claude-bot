## 機能

### コア機能

- **WeChatリモート操作** — WeChatメッセージからローカルのClaude Codeを直接操作、PCのターミナルを開く必要なし
- **WeChat終了後も動作** — サーバーサイドのロングポーリングにより、WeChatアプリを閉じてもメッセージはキューに入ります。再度開けば返信を確認可能。Bot は24時間365日稼働
- **完全なPC操作能力** — Claude Codeはコンピュータを本格的に操作可能：任意のファイル読み書き、ターミナルコマンド実行、コード検索、パッケージインストール、Git操作など。単なるチャットではありません
- **リアルプロジェクトを直接操作** — Claude Codeはローカルプロジェクトディレクトリで実行、実際のコードファイルを直接変更、即座に反映

### ユーザー体験

- **リアルタイム進捗** — Claude Code動作中にWeChatへライブ進捗を通知（例：`📖 ファイル読取中: src/app.js`）
- **タイピング表示** — 処理中にWeChat上で「入力中」を表示
- **スマートメッセージ分割** — 長い返信はコードブロック境界で自動分割、シーケンス番号付き
- **Markdown変換** — Markdown出力をWeChat向けプレーンテキストに自動変換
- **画像/ファイル/動画対応** — WeChatから送られた画像・ファイル・動画を受信、自動ダウンロードしてClaude Codeで分析；`/send` で本機ファイルを送り返し
- **音声メッセージ対応** — WeChat音声テキスト変換に対応、音声でもコマンド送信可能
- **スラッシュコマンド** — `/new` 会話リセット、`/model` モデル切替、`/send` ファイル送信、`/status` 状態確認

### 安定性とセキュリティ

- **マルチユーザーセッション分離** — 各ユーザー独立セッション、会話コンテキスト継続
- **同時実行制御** — 最大3タスク同時処理、超過分は自動キュー、メッセージ損失なし
- **自動再接続** — セッション期限切れ時に自動再ログイン、トークン永続化
- **セッション期限切れクリーンアップ** — 1時間非アクティブなセッションは自動回収、最大100セッション保持
- **段階的タイムアウト保護** — 2分間無応答でリマインダー、5分で強制終了、タスク停滞を防止
- **グレースフルシャットダウン** — Ctrl+Cで安全終了、子プロセス自動クリーンアップ
- **ローカル実行** — コードとデータはサードパーティサーバーを経由せず、資格情報ファイルはオーナーのみ読み書き可能

## なぜOpenClawを直接使わないのか？

WeChat公式の[OpenClaw](https://github.com/nicepkg/openclaw)は、複数のモデルバックエンドに対応し、ClawBotプロトコルでWeChatに接続する多機能AIエージェントフレームワークです。カスタマイズ可能なAI機能が必要なシナリオに適した完全なプラットフォームソリューションです。

ただし、**WeChatからローカルのClaude Codeをリモート操作する**ことが目的なら、OpenClawは最適解ではありません。以下が核心的な違いです：

### 軽量 vs 重量級

| 比較項目 | OpenClaw | 本プロジェクト (WeChat Claude Code Bot) |
|---------|----------|---------------------------------------|
| **位置づけ** | 汎用AIエージェントフレームワーク、複数モデル・プラグイン対応 | 一つのことに集中：WeChatとClaude Code CLIの接続 |
| **インストール** | フレームワーク + APIキー + モデル設定 + プラグインシステム + 依存チェーン | **3ステップ**：`git clone` → `npm install` → `npm start` |
| **依存関係** | フレームワーク巨大、依存多数、インストールで問題が起きやすい | **依存2つのみ**（dotenv + qrcode-terminal） |
| **メンテナンス** | フレームワーク更新頻繁、バージョン互換性に問題あり | ほぼメンテナンス不要、コードはシンプルで透明 |
| **コード量** | 完全なフレームワーク、数千ファイル | **コアファイル4つ**、合計約1000行 |

### トークンコスト

これが最も重要な違いです：

| | OpenClaw | 本プロジェクト |
|-|----------|-------------|
| **課金方式** | 会話ごとにClaude APIを呼び出し、トークン単位で課金 | ローカルClaude Code CLIを使用、サブスクリプション枠で動作 |
| **コスト特性** | 長い会話、コード分析、多段階のやり取りでトークン消費大、費用が予測不能 | **追加トークン費用ゼロ**、既存のClaude Codeサブスクリプションがあればそれだけで利用可能 |
| **APIキー** | 必須 | 不要 |

### Claude Codeの独自機能

本プロジェクトがAPIを直接呼ぶのではなくClaude Code CLIに接続する理由は、Claude Codeが**API呼び出しでは代替できない機能**を提供するからです：

- **完全なPC操作** — 任意のファイル読み書き、任意のターミナルコマンド実行、コードベース全体の検索。単なるテキスト会話ではない
- **プロジェクトレベルのコンテキスト** — プロジェクト全体がClaude Codeのコンテキスト、コードベース構造全体を理解
- **10以上の内蔵ツール** — Read、Write、Edit、Bash、Glob、Grep、WebSearch等、完全なツールチェーン
- **実際のコード操作** — 実ファイルを直接修正、Git操作実行、依存関係インストール、変更は即座に反映
- **リアルタイム進捗** — すべての操作（ファイル読取、コマンド実行、コード編集）をリアルタイムでWeChatに通知
- **セッション継続** — 独立セッション管理、会話コンテキストがターンを超えて持続

これらの機能はAPI呼び出し+独自ツールチェーン構築でも実現可能ですが、Claude Code CLIはすでに完成された成熟製品です。車輪の再発明は不要です。

### 一言でまとめると

> **OpenClaw** = 汎用シナリオ向けの多機能AIフレームワーク。だが重い、高い、セットアップが大変
>
> **本プロジェクト** = ファイル4つ、追加コストゼロ、一つのことに集中：WeChatからClaude Codeをリモート操作

### 選び方

- **汎用AIのWeChat Botを構築したい**、複数モデル・カスタムプラグイン対応 → OpenClaw
- **WeChatからPC操作・コード修正・コマンド実行をしたい** → 本プロジェクト
- **Claude Codeのサブスクリプションを持っている**、追加コストゼロでWeChat遠隔操作を実現したい → 本プロジェクト

## 仕組み

```
┌──────────┐         ┌──────────────────┐         ┌─────────────┐
│  WeChat   │ ─メッセージ▶│  iLink Bot API   │ ─ポーリング▶│ ローカルBot  │
│ (スマホ)   │ ◀──返信── │ (weixin.qq.com)  │ ◀──送信── │  (Node.js)   │
└──────────┘         └──────────────────┘         └──────┬──────┘
                                                         │
                                                         │ CLI呼出
                                                         ▼
                                                   ┌───────────┐
                                                   │ Claude Code│
                                                   │ (ローカル)  │
                                                   └───────────┘
```

1. BotがWeChat iLink Bot API（ロングポーリング）でユーザーメッセージを受信
2. メッセージをローカルのClaude Code CLI（stream-jsonモード）に転送
3. Claude Codeのツール呼び出し（ファイル読取、コード記述、コマンド実行等）をリアルタイムで解析し、進捗をWeChatに送信
4. Claude Code完了後、最終結果をフォーマットしてWeChatに送信

## クイックスタート

### 前提条件

- **Node.js** >= 18
- **Claude Code CLI** がグローバルインストール済み（`npm install -g @anthropic-ai/claude-code`）
- **WeChatアカウント**

### インストール手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/mrliuzhiyu/Wechat-Claude-bot.git
cd Wechat-Claude-bot

# 2. 依存関係をインストール
npm install

# 3. （オプション）作業ディレクトリを設定
cp .env.example .env
# .env を編集して CLAUDE_CWD をプロジェクトパスに設定

# 4. Botを起動
npm start
```

### 初回接続

1. 起動後、ターミナルにQRコードが表示される
2. WeChatを開く → QRコードをスキャン
3. WeChatで接続を確認
4. `✅ 接続成功` が表示されたら使用可能
5. WeChatでBotにメッセージを送信するだけ

> 初回ログイン後、トークンは自動的に保存されます。次回起動時にスキャンし直す必要はありません（トークン期限切れを除く）。

## 設定

`.env` ファイルまたは環境変数で設定：

| 変数 | 説明 | デフォルト |
|------|------|-----------|
| `CLAUDE_CWD` | Claude Codeの作業ディレクトリ | カレントディレクトリ (`process.cwd()`) |
| `SYSTEM_PROMPT` | 追加のシステムプロンプト | 空 |

**`.env` ファイルの例：**

```bash
# Claude Codeの操作対象プロジェクトディレクトリを指定
CLAUDE_CWD=/home/user/my-project

# カスタムシステムプロンプト（オプション）
SYSTEM_PROMPT=あなたはReact開発に特化したアシスタントです
```

## 使い方ガイド

### 基本的な使い方

WeChatで自然言語でニーズを記述するだけ。Claude Codeが自動的に実行します：

```
あなた: プロジェクト構造を見せて
Bot: 🤖 受信しました、処理中...
Bot: 🔍 ファイル検索中: **/*
Bot: プロジェクト構造：
     ├── src/
     │   ├── components/
     │   ├── pages/
     │   └── utils/
     ├── package.json
     └── README.md
```

### リアルタイム進捗

Claude Codeが操作を実行すると、リアルタイムで進捗フィードバックを受け取れます：

```
📖 ファイル読取中: src/app.js
✏️ ファイル編集中: src/utils.js
⚡ コマンド実行中: npm test
🔍 ファイル検索中: **/*.ts
🔍 コンテンツ検索中: handleClick
📝 ファイル作成中: src/helper.js
📋 タスク計画中
```

### 長いメッセージの処理

Claude Codeの返信が4000文字を超えると、メッセージがスマートに分割されます：

- コードブロック境界での分割を優先
- 次に空行での分割
- 各チャンクにシーケンス番号を表示、例：`(続き 2/3)`

## コマンド一覧

| コマンド | 説明 |
|---------|------|
| `/help` | ヘルプ情報を表示 |
| `/new` | 会話をリセット、新しいセッションを開始 |
| `/model` | モデル切替（sonnet / opus / haiku） |
| `/send <パス>` | 本機のファイルをWeChatに送信（画像、ファイル等） |

## よくある質問

### Q: 起動時に「claudeコマンドが見つかりません」

Claude Code CLIがグローバルインストールされていることを確認：

```bash
npm install -g @anthropic-ai/claude-code
```

`claude --version` で確認してください。

### Q: QRコードが正しく表示されない

ターミナルがUnicodeをサポートしていない場合、QRコードが正しく表示されないことがあります。起動ログにURLが含まれています。ブラウザで開いてスキャンしてください。

### Q: トークンが期限切れになった場合は？

Botはトークンの期限切れを自動検出し、新しいQRコードを表示します。手動操作は不要です。

### Q: 複数人で同時に使えますか？

はい。各WeChatユーザーは独立したセッションを持ちます。最大3つの同時リクエストをサポートし、超過分は自動的にキューに入ります。

### Q: リクエストがタイムアウトした

デフォルトのタイムアウトはリクエストあたり5分です。複雑なタスクの場合、小さなステップに分割してください。例：まずClaude にプロジェクト構造を探索させ、その後具体的な操作を実行。

### Q: 画像/ファイルの送信に対応していますか？

はい。BotはWeChatから送られた画像・ファイル・動画を受信し、自動的にローカルにダウンロードしてClaude Codeに分析させます。`/send <ファイルパス>` で本機のファイルをWeChatに送り返すことも可能です。音声メッセージにはWeChatの音声テキスト変換機能が必要です。

### Q: セキュリティはどうですか？

- Botはローカルマシンで実行され、コードはサードパーティサーバーを経由しません
- Claude Codeは`bypassPermissions`モードで実行され、完全なファイルとコマンドアクセス権を持ちます
- ログイン資格情報はローカルの`.state/`にオーナーのみ読み書き可能な権限で保存されます
- `.env`は`.gitignore`に含まれており、Gitにコミットされません

> **注意**：Claude Codeは完全な権限を持つため、信頼できる人だけがBotにメッセージを送信できるようにしてください。
