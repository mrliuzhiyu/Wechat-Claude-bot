## 機能

- **WeChatリモート操作** — WeChatメッセージからローカルのClaude Codeを直接操作、PCのターミナルを開く必要なし
- **WeChat終了後も動作** — サーバーサイドのロングポーリングにより、WeChatアプリを閉じてもメッセージはキューに入ります。再度開けば返信を確認可能。Bot は24時間365日稼働
- **リアルタイム進捗** — Claude Codeがファイル読取・コード記述・コマンド実行時、リアルタイムでWeChatに進捗を通知
- **完全なコード操作能力** — Claude Codeは完全な権限を持つ：ファイル読み書き、ターミナルコマンド実行、コード検索、パッケージインストール
- **マルチユーザーセッション分離** — 各WeChatユーザーは独立したセッションを持ち、会話コンテキストが継続、相互に干渉しない
- **音声メッセージ対応** — WeChat音声メッセージに対応（音声テキスト変換の有効化が必要）
- **スマートメッセージ分割** — 長い返信はコードブロック境界で自動的に分割、コードの途中で切れることなし
- **Markdown変換** — Claudeの Markdown出力をWeChat向けプレーンテキストに自動変換
- **自動再接続** — セッション期限切れ時に自動再ログイン、トークン永続化で再スキャン不要
- **同時実行制御** — 最大3タスクを同時処理、超過分は自動キュー
- **グレースフルシャットダウン** — Ctrl+Cで安全終了、子プロセスを自動クリーンアップ

## 仕組み

```
┌──────────┐         ┌──────────────────┐         ┌─────────────┐
│  WeChat   │ ─メッセージ▶│  iLink Bot API   │ ─ポーリング▶│ ローカルBot  │
│ (スマホ)   │ ◀──返信── │ (weixin.qq.com)  │ ◀──送信── │  (Node.js)   │
└──────────┘         └──────────────────┘         └──────┬──────┘
                                                         │
                                                         │ CLI呼出
                                                         ▼
                                                   ┌───────────┐
                                                   │ Claude Code│
                                                   │ (ローカル)  │
                                                   └───────────┘
```

1. BotがWeChat iLink Bot API（ロングポーリング）でユーザーメッセージを受信
2. メッセージをローカルのClaude Code CLI（stream-jsonモード）に転送
3. Claude Codeのツール呼び出し（ファイル読取、コード記述、コマンド実行等）をリアルタイムで解析し、進捗をWeChatに送信
4. Claude Code完了後、最終結果をフォーマットしてWeChatに送信

## クイックスタート

### 前提条件

- **Node.js** >= 18
- **Claude Code CLI** がグローバルインストール済み（`npm install -g @anthropic-ai/claude-code`）
- **WeChatアカウント**

### インストール手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/mrliuzhiyu/Wechat-Claude-bot.git
cd Wechat-Claude-bot

# 2. 依存関係をインストール
npm install

# 3. （オプション）作業ディレクトリを設定
cp .env.example .env
# .env を編集して CLAUDE_CWD をプロジェクトパスに設定

# 4. Botを起動
npm start
```

### 初回接続

1. 起動後、ターミナルにQRコードが表示される
2. WeChatを開く → QRコードをスキャン
3. WeChatで接続を確認
4. `✅ 接続成功` が表示されたら使用可能
5. WeChatでBotにメッセージを送信するだけ

> 初回ログイン後、トークンは自動的に保存されます。次回起動時にスキャンし直す必要はありません（トークン期限切れを除く）。

## 設定

`.env` ファイルまたは環境変数で設定：

| 変数 | 説明 | デフォルト |
|------|------|-----------|
| `CLAUDE_CWD` | Claude Codeの作業ディレクトリ | カレントディレクトリ (`process.cwd()`) |
| `SYSTEM_PROMPT` | 追加のシステムプロンプト | 空 |

**`.env` ファイルの例：**

```bash
# Claude Codeの操作対象プロジェクトディレクトリを指定
CLAUDE_CWD=/home/user/my-project

# カスタムシステムプロンプト（オプション）
SYSTEM_PROMPT=あなたはReact開発に特化したアシスタントです
```

## 使い方ガイド

### 基本的な使い方

WeChatで自然言語でニーズを記述するだけ。Claude Codeが自動的に実行します：

```
あなた: プロジェクト構造を見せて
Bot: 🤖 受信しました、処理中...
Bot: 🔍 ファイル検索中: **/*
Bot: プロジェクト構造：
     ├── src/
     │   ├── components/
     │   ├── pages/
     │   └── utils/
     ├── package.json
     └── README.md
```

### Claude Codeにできること

WeChatメッセージを通じて、Claude Codeに以下の操作ができます：

- **コード読取** — 「src/app.jsの内容を見せて」
- **コード作成** — 「utilsディレクトリに日付フォーマット関数を作成して」
- **コード修正** — 「Appコンポーネントの背景色を青に変更して」
- **コマンド実行** — 「npm testを実行して結果を見せて」
- **コード検索** — 「useStateを使用している箇所を全て検索して」
- **パッケージインストール** — 「axiosとlodashをインストールして」
- **デバッグ** — 「なぜビルドが失敗しているの？確認して」
- **コードレビュー** — 「最近の変更に問題がないかレビューして」
- **Git操作** — 「現在の変更をコミットして、メッセージは'ログインバグ修正'」

### リアルタイム進捗

Claude Codeが操作を実行すると、リアルタイムで進捗フィードバックを受け取れます：

```
📖 ファイル読取中: src/app.js
✏️ ファイル編集中: src/utils.js
⚡ コマンド実行中: npm test
🔍 ファイル検索中: **/*.ts
🔍 コンテンツ検索中: handleClick
📝 ファイル作成中: src/helper.js
📋 タスク計画中
```

### 長いメッセージの処理

Claude Codeの返信が4000文字を超えると、メッセージがスマートに分割されます：

- コードブロック境界での分割を優先
- 次に空行での分割
- 各チャンクにシーケンス番号を表示、例：`(続き 2/3)`

## よくある質問

### Q: 起動時に「claudeコマンドが見つかりません」

Claude Code CLIがグローバルインストールされていることを確認：

```bash
npm install -g @anthropic-ai/claude-code
```

`claude --version` で確認してください。

### Q: QRコードが正しく表示されない

ターミナルがUnicodeをサポートしていない場合、QRコードが正しく表示されないことがあります。起動ログにURLが含まれています。ブラウザで開いてスキャンしてください。

### Q: トークンが期限切れになった場合は？

Botはトークンの期限切れを自動検出し、新しいQRコードを表示します。手動操作は不要です。

### Q: 複数人で同時に使えますか？

はい。各WeChatユーザーは独立したセッションを持ちます。最大3つの同時リクエストをサポートし、超過分は自動的にキューに入ります。

### Q: リクエストがタイムアウトした

デフォルトのタイムアウトはリクエストあたり5分です。複雑なタスクの場合、小さなステップに分割してください。例：まずClaude にプロジェクト構造を探索させ、その後具体的な操作を実行。

### Q: 画像/ファイルの送信に対応していますか？

現在、テキストメッセージと音声メッセージ（音声テキスト変換有効時）のみ対応しています。画像、動画、ファイルは未対応です。

### Q: セキュリティはどうですか？

- Botはローカルマシンで実行され、コードはサードパーティサーバーを経由しません
- Claude Codeは`bypassPermissions`モードで実行され、完全なファイルとコマンドアクセス権を持ちます
- ログイン資格情報はローカルの`.state/`にオーナーのみ読み書き可能な権限で保存されます
- `.env`は`.gitignore`に含まれており、Gitにコミットされません

> **注意**：Claude Codeは完全な権限を持つため、信頼できる人だけがBotにメッセージを送信できるようにしてください。
