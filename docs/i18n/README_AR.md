## الميزات

### القدرات الأساسية

- **التحكم عن بُعد عبر WeChat** — تحكم في Claude Code المحلي مباشرة عبر رسائل WeChat، دون الحاجة لفتح الطرفية
- **يعمل بعد إغلاق WeChat** — بفضل الاستطلاع الطويل من جانب الخادم، تُوضع الرسائل في قائمة الانتظار حتى بعد إغلاق تطبيق WeChat. أعد فتح WeChat لرؤية الردود — البوت يعمل على مدار الساعة طوال أيام الأسبوع
- **تحكم كامل بالحاسوب** — يمكن لـ Claude Code تشغيل حاسوبك فعلياً: قراءة/كتابة الملفات، تنفيذ الأوامر، البحث في الكود، تثبيت الحزم، عمليات Git — ليس مجرد محادثة
- **العمل على مشاريع حقيقية** — يعمل Claude Code في دليل مشروعك المحلي، يعدّل ملفات الكود الحقيقية مباشرة

### تجربة المستخدم

- **تقدم فوري** — تحديثات مباشرة في WeChat أثناء عمل Claude Code (مثل: `📖 جارٍ قراءة الملف: src/app.js`)
- **مؤشر الكتابة** — يعرض حالة "يكتب" في WeChat أثناء المعالجة
- **تقسيم ذكي للرسائل** — الردود الطويلة تُقسم عند حدود كتل الكود مع ترقيم
- **تحويل Markdown** — تحويل تلقائي لـ Markdown إلى نص متوافق مع WeChat
- **صور/ملفات/فيديو** — استقبال الصور والملفات والفيديو من WeChat، تنزيل تلقائي وتحليل عبر Claude Code؛ `/send` لإرسال الملفات المحلية
- **رسائل صوتية** — دعم تحويل الصوت إلى نص في WeChat
- **أوامر السلاش** — `/new` إعادة تعيين، `/model` تبديل النموذج، `/send` إرسال ملف، `/status` الحالة

### الاستقرار والأمان

- **عزل الجلسات** — كل مستخدم يحصل على جلسة مستقلة مع سياق مستمر
- **التحكم في التزامن** — حتى 3 مهام متزامنة، الزائد في قائمة الانتظار بدون فقدان
- **إعادة اتصال تلقائية** — إعادة مصادقة تلقائية عند انتهاء الجلسة
- **تنظيف الجلسات** — الجلسات غير النشطة تُنظف بعد ساعة، حد أقصى 100 جلسة
- **حماية متدرجة من التوقف** — تذكير بعد دقيقتين، إنهاء إجباري بعد 5 دقائق
- **إيقاف سلس** — Ctrl+C مع تنظيف تلقائي للعمليات الفرعية
- **تشغيل محلي** — الكود والبيانات لا تمر عبر خوادم طرف ثالث

## لماذا لا تستخدم OpenClaw مباشرة؟

[OpenClaw](https://github.com/nicepkg/openclaw) الرسمي من WeChat هو إطار عمل شامل لعملاء الذكاء الاصطناعي، يدعم نماذج وإضافات متعددة. حل على مستوى المنصة للسيناريوهات التي تحتاج قدرات ذكاء اصطناعي قابلة للتخصيص.

لكن إذا كان هدفك هو **التحكم في Claude Code المحلي عبر WeChat**، فإن OpenClaw ليس الخيار الأمثل:

### خفيف vs ثقيل

| البُعد | OpenClaw | هذا المشروع |
|--------|----------|-------------|
| **الغرض** | إطار عمل ذكاء اصطناعي شامل، نماذج وإضافات متعددة | شيء واحد: ربط WeChat بـ Claude Code CLI |
| **التثبيت** | إطار العمل + مفتاح API + إعدادات + إضافات + تبعيات | **3 خطوات**: `git clone` → `npm install` → `npm start` |
| **التبعيات** | إطار عمل ضخم، تبعيات كثيرة | **تبعيتان فقط** (dotenv + qrcode-terminal) |
| **الصيانة** | تحديثات متكررة، مشاكل توافق | شبه معدومة، كود بسيط وشفاف |
| **الكود** | آلاف الملفات | **4 ملفات أساسية**، حوالي 1000 سطر |

### تكلفة التوكن

| | OpenClaw | هذا المشروع |
|-|----------|-------------|
| **الفوترة** | يستدعي Claude API لكل محادثة، فوترة بالتوكن | Claude Code CLI محلي، حصة الاشتراك |
| **التكلفة** | المحادثات الطويلة وتحليل الكود تحرق التوكن بسرعة | **صفر تكلفة توكن إضافية** |
| **مفتاح API** | مطلوب | غير مطلوب |

### قدرات Claude Code الفريدة

يوفر Claude Code CLI **قدرات لا يمكن لاستدعاءات API وحدها تكرارها**: تحكم كامل بالحاسوب، سياق المشروع بالكامل، أكثر من 10 أدوات مدمجة (Read, Write, Edit, Bash, Glob, Grep...)، عمليات كود حقيقية مع تأثير فوري، تقدم فوري، وجلسات مستمرة.

### بجملة واحدة

> **OpenClaw** = إطار عمل ذكاء اصطناعي شامل للسيناريوهات العامة — لكنه ثقيل ومكلف ومعقد التثبيت
>
> **هذا المشروع** = 4 ملفات، تكلفة صفر إضافية، هدف واحد: التحكم بـ Claude Code عبر WeChat

### كيف تختار؟

- **بوت ذكاء اصطناعي عام لـ WeChat** بنماذج متعددة → OpenClaw
- **التحكم بالحاسوب والكود والأوامر عبر WeChat** → هذا المشروع
- **لديك اشتراك Claude Code** → هذا المشروع (تكلفة إضافية صفر)

## كيف يعمل

```
┌──────────┐         ┌──────────────────┐         ┌───────────┐
│  WeChat   │ ─رسالة─▶│  iLink Bot API   │ ─استطلاع▶│ بوت محلي  │
│ (الهاتف)  │ ◀─رد──── │ (weixin.qq.com)  │ ◀─إرسال── │ (Node.js) │
└──────────┘         └──────────────────┘         └─────┬─────┘
                                                        │
                                                        │ استدعاء CLI
                                                        ▼
                                                  ┌───────────┐
                                                  │ Claude Code│
                                                  │  (محلي)    │
                                                  └───────────┘
```

1. يستقبل البوت رسائل المستخدمين عبر WeChat iLink Bot API (استطلاع طويل)
2. يُعيد توجيه الرسائل إلى Claude Code CLI المحلي (وضع stream-json)
3. يُحلل استدعاءات أدوات Claude Code في الوقت الفعلي، ويُرسل التقدم إلى WeChat
4. بعد الانتهاء، يُنسق النتيجة النهائية ويُرسلها إلى WeChat

## البدء السريع

### المتطلبات المسبقة

- **Node.js** >= 18
- **Claude Code CLI** مُثبت عالمياً (`npm install -g @anthropic-ai/claude-code`)
- **حساب WeChat**

### التثبيت

```bash
# 1. استنساخ المستودع
git clone https://github.com/mrliuzhiyu/Wechat-Claude-bot.git
cd Wechat-Claude-bot

# 2. تثبيت التبعيات
npm install

# 3. (اختياري) إعداد دليل العمل
cp .env.example .env
# حرر .env لتعيين CLAUDE_CWD إلى مسار مشروعك

# 4. تشغيل البوت
npm start
```

### الاتصال الأول

1. بعد التشغيل، سيظهر رمز QR في الطرفية
2. افتح WeChat ← امسح رمز QR
3. أكد الاتصال في WeChat
4. عند ظهور `✅ تم الاتصال!`، البوت جاهز
5. أرسل رسالة للبوت في WeChat للبدء

> بعد أول تسجيل دخول، يُحفظ الرمز تلقائياً. لا حاجة لإعادة المسح عند التشغيل التالي (إلا إذا انتهت صلاحية الرمز).

## الإعدادات

الإعداد عبر ملف `.env` أو متغيرات البيئة:

| المتغير | الوصف | القيمة الافتراضية |

| `CLAUDE_CWD` | دليل عمل Claude Code | الدليل الحالي (`process.cwd()`) |

**مثال ملف `.env`:**

```bash

SYSTEM_PROMPT=أنت مساعد متخصص في تطوير React
```

## دليل الاستخدام

### الاستخدام الأساسي

أرسل رسائل بلغة طبيعية في WeChat تصف ما تحتاجه. سيُنفذ Claude Code تلقائياً:

```
أنت: اعرض هيكل المشروع
Bot: 🤖 تم الاستلام، جارٍ المعالجة...
Bot: 🔍 جارٍ البحث عن الملفات: **/*
Bot: هيكل المشروع:
     ├── src/
     │   ├── components/
     │   ├── pages/
     │   └── utils/
     ├── package.json
     └── README.md
```

### ما يمكن لـ Claude Code فعله

عبر رسائل WeChat، يمكنك طلب:

- **قراءة الكود** — "اعرض محتوى src/app.js"
- **كتابة الكود** — "أنشئ دالة تنسيق التاريخ في utils"
- **تعديل الكود** — "غيّر لون خلفية مكون App إلى الأزرق"
- **تنفيذ الأوامر** — "شغّل npm test واعرض النتائج"
- **البحث في الكود** — "ابحث عن جميع الأماكن التي تستخدم useState"
- **تثبيت الحزم** — "ثبّت axios و lodash"
- **تصحيح الأخطاء** — "لماذا يفشل البناء؟ تحقق"
- **مراجعة الكود** — "راجع التغييرات الأخيرة بحثاً عن مشاكل"
- **عمليات Git** — "قم بعمل commit برسالة 'fix: إصلاح خطأ تسجيل الدخول'"

### التقدم الفوري

عند تنفيذ Claude Code للعمليات، تتلقى تحديثات فورية:

```
📖 جارٍ قراءة الملف: src/app.js
✏️ جارٍ تحرير الملف: src/utils.js
⚡ جارٍ تنفيذ الأمر: npm test
🔍 جارٍ البحث عن الملفات: **/*.ts
🔍 جارٍ البحث في المحتوى: handleClick
📝 جارٍ إنشاء الملف: src/helper.js
📋 جارٍ تخطيط المهام
```

### معالجة الرسائل الطويلة

عندما يتجاوز رد Claude Code 4000 حرف، تُقسم الرسائل بذكاء:

- الأولوية للتقسيم عند حدود كتل الكود
- ثم عند الأسطر الفارغة
- كل جزء يُوسم برقم تسلسلي، مثل: `(تابع 2/3)`

## حالات الاستخدام

### الحالة 1: إصلاح خطأ أثناء التنقل

> زميل يُبلغ عن خطأ عاجل في الإنتاج وأنت في المترو.

```
أنت: اعرض دالة login في src/api/auth.js
Bot: [يعرض الكود]

أنت: التحقق من الرمز في السطر 42 خاطئ، يجب أن يكون > وليس >=
Bot: ✏️ جارٍ تحرير الملف: api/auth.js
Bot: تم الإصلاح، تم تغيير >= إلى >

أنت: شغّل الاختبارات
Bot: ⚡ جارٍ تنفيذ: npm test
Bot: جميع الاختبارات الـ 23 نجحت ✓

أنت: قم بعمل commit "fix: تصحيح شرط حد انتهاء صلاحية الرمز"
Bot: تم عمل commit ودفعه إلى المستودع البعيد
```

### الحالة 2: تطوير ميزة من الهاتف

> عطلة نهاية الأسبوع، خارج المنزل، فكرة سريعة.

```
أنت: أنشئ مكون ThemeToggle في src/components يدعم الوضع الداكن/الفاتح
Bot: 📝 جارٍ إنشاء: components/ThemeToggle.jsx
Bot: ✏️ جارٍ تحرير: App.jsx
Bot: تم إنشاء مكون ThemeToggle واستيراده في App.jsx...
```

### الحالة 3: مراجعة الكود والتعلم

> انضممت لمشروع جديد وتريد فهم قاعدة الكود بسرعة.

```
أنت: ما البنية العامة لهذا المشروع؟
Bot: [يحلل الهيكل، الوحدات الرئيسية، المكدس التقني...]

أنت: أين منطق الاتصال بقاعدة البيانات؟
Bot: 🔍 جارٍ البحث: database|connection|mongoose
Bot: اتصال قاعدة البيانات موجود في src/config/db.js...
```

### الحالة 4: DevOps والمراقبة

> تحتاج التحقق من حالة الخدمة وأنت خارج المكتب.

```
أنت: تحقق من حالة حاويات Docker
Bot: ⚡ جارٍ تنفيذ: docker ps
Bot: [يعرض قائمة الحاويات...]

أنت: تحقق من السجلات الأخيرة بحثاً عن أخطاء
Bot: ⚡ جارٍ تنفيذ: docker logs --tail 50 my-app
Bot: [يعرض السجلات...]
```

## الأسئلة الشائعة

### س: "أمر claude غير موجود" عند التشغيل

تأكد من تثبيت Claude Code CLI عالمياً:

```bash
npm install -g @anthropic-ai/claude-code
```

تحقق بـ `claude --version`.

### س: رمز QR لا يُعرض بشكل صحيح

إذا لم تدعم الطرفية Unicode، قد لا يُعرض رمز QR بشكل صحيح. سجل التشغيل يتضمن رابط URL — افتحه في المتصفح للمسح.

### س: ماذا لو انتهت صلاحية الرمز؟

يكتشف البوت انتهاء الصلاحية تلقائياً ويعرض رمز QR جديد. لا حاجة لإجراء يدوي.

### س: هل يمكن لعدة أشخاص استخدامه في وقت واحد؟

نعم. كل مستخدم WeChat يحصل على جلسة مستقلة. يدعم حتى 3 طلبات متزامنة؛ الزائد يُوضع في قائمة الانتظار تلقائياً.

### س: انتهت مهلة الطلب

المهلة الافتراضية 5 دقائق لكل طلب. للمهام المعقدة، قسّمها إلى خطوات أصغر — مثلاً، اطلب أولاً من Claude استكشاف هيكل المشروع، ثم نفّذ عمليات محددة.

### س: هل يدعم الصور/الملفات؟

نعم. يمكن للبوت استقبال الصور والملفات والفيديو من WeChat، وتنزيلها تلقائياً محلياً وتمريرها لـ Claude Code للتحليل. يمكنك أيضاً استخدام `/send <مسار>` لإرسال ملفات محلية إلى WeChat. الرسائل الصوتية تتطلب ميزة تحويل الصوت إلى نص.

### س: ما مدى الأمان؟

- يعمل البوت محلياً على جهازك — الكود لا يمر عبر خوادم طرف ثالث
- يعمل Claude Code في وضع `bypassPermissions` مع وصول كامل للملفات والأوامر
- بيانات الاعتماد تُخزن محلياً في `.state/` بصلاحيات المالك فقط
- `.env` مُدرج في `.gitignore` ولن يُرفع إلى Git

> **تحذير**: بما أن Claude Code يملك صلاحيات كاملة، تأكد أن الأشخاص الموثوقين فقط يمكنهم إرسال رسائل للبوت.
